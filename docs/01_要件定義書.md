# 要件定義書

**プロジェクト名:** 日本ハザードマネージャー  
**バージョン:** 1.0  
**作成日:** 2026-06-02

---

## 1. プロジェクト概要

### 1.1 目的

日本全国で発生する災害・軍事攻撃などのハザード情報を一元管理し、地図上でリアルタイムに可視化するWebアプリケーションを提供する。複数のオペレータが同時にアクセスし、ハザードの登録・更新・削除を行える環境を実現する。

### 1.2 背景

ハザード情報の管理が分散していると、緊急時の対応が遅延する。本システムは情報を一元化し、状況の迅速な把握と意思決定を支援する。

### 1.3 用語定義

| 用語 | 定義 |
|------|------|
| ハザード | 災害・軍事攻撃等、危険を伴う事象 |
| 危険レベル | ハザードの深刻度を表す1〜5の整数値 |
| ステータス | ハザードの現在状態（発生中 / 監視中 / 解除済） |
| オペレータ | 本システムを操作する担当者 |

---

## 2. 利用者・利用環境

### 2.1 利用者

- **ハザード管理オペレータ:** ハザードの登録・編集・削除を行う担当者
- **状況確認者:** 地図・一覧でハザード情報を参照する担当者

### 2.2 利用環境

- **クライアント:** モダンブラウザ（Chrome / Firefox / Edge）
- **ネットワーク:** インターネット接続必須（地図タイル・逆ジオコーディング使用）
- **サーバー:** Docker が動作する Linux 環境

---

## 3. 機能要件

### 3.1 地図表示機能

| ID | 要件 |
|----|------|
| F-01 | 日本地図（OpenStreetMap）を初期表示し、緯度 37°・経度 137°・ズーム 6 を初期視点とする |
| F-02 | 登録済みハザードを危険レベルに応じた色の円形ピンで地図上に表示する |
| F-03 | ステータスが「解除済」のピンはグレー（#6b7280）で表示する |
| F-04 | ピンにカーソルを合わせるとタイトル・種別・ステータスをツールチップ表示する |
| F-05 | ピンをクリックするとハザード詳細パネルを表示する |
| F-06 | 地図上の空白エリアをクリックするとハザード新規登録フォームを表示する |

### 3.2 ハザード登録機能

| ID | 要件 |
|----|------|
| F-10 | 地図クリック時の座標を緯度・経度として自動セットする |
| F-11 | クリック座標から逆ジオコーディングにより都道府県・市区町村を自動入力する |
| F-12 | 以下の情報を入力して登録できる：種別、タイトル（必須）、詳細説明、危険レベル（1〜5）、場所、発生日時 |
| F-13 | 登録完了後、地図・一覧に即時反映する |

### 3.3 ハザード詳細・編集機能

| ID | 要件 |
|----|------|
| F-20 | 詳細パネルに以下を表示する：タイトル、種別、危険レベル、ステータス、説明、座標、場所、発生日時、解除日時 |
| F-21 | 詳細パネルからハザードの全フィールドを編集できる（座標を除く） |
| F-22 | ステータスをワンクリックで変更できる（発生中 / 監視中 / 解除済） |
| F-23 | 編集保存時に楽観的ロックにより競合を検知する（[CF-01] 参照） |

### 3.4 ハザード削除機能

| ID | 要件 |
|----|------|
| F-30 | 一覧のゴミ箱アイコンをクリックして削除できる |
| F-31 | 削除前に確認ダイアログを表示する |
| F-32 | 削除後、地図・一覧から即時除去する |

### 3.5 ハザード一覧・フィルタ機能

| ID | 要件 |
|----|------|
| F-40 | 画面左サイドバーにハザード一覧を発生日時の降順で表示する |
| F-41 | 種別・ステータス・危険レベル（以上）でフィルタリングできる |
| F-42 | 一覧から選択するとハザード詳細パネルを開く |

### 3.6 リアルタイム同期機能

| ID | 要件 |
|----|------|
| F-50 | 30秒ごとにサーバーから最新のハザード情報を取得し画面に反映する |
| F-51 | 別ユーザーによる変更を自動で取り込む（編集中の場合は取り込まない） |

---

## 4. 競合制御要件

| ID | 要件 |
|----|------|
| CF-01 | 複数ユーザーが同一ハザードを同時編集した場合、後から保存しようとしたユーザーに競合を通知する |
| CF-02 | 競合通知時、「最新版を確認する」または「強制上書き保存」を選択できる |
| CF-03 | 他ユーザーに削除されたハザードを保存しようとした場合、エラーを通知してパネルを閉じる |

---

## 5. 非機能要件

### 5.1 パフォーマンス

| ID | 要件 |
|----|------|
| NF-01 | 地図の初期表示を3秒以内に完了する |
| NF-02 | API レスポンスタイムを200ms以内とする（通常負荷時） |

### 5.2 信頼性

| ID | 要件 |
|----|------|
| NF-10 | ポーリング中の通信エラーは無視し、次のポーリング周期で再試行する |
| NF-11 | DB（SQLite）はコンテナ外のファイルとして永続化し、再起動後もデータを保持する |

### 5.3 ユーザビリティ

| ID | 要件 |
|----|------|
| NF-20 | UIは日本語で表示する |
| NF-21 | 地図上のカーソルは通常時に矢印、ドラッグ中に手のひらアイコンを表示する |
| NF-22 | 非同期処理中はローディング状態を明示する |

### 5.4 保守性

| ID | 要件 |
|----|------|
| NF-30 | Docker Compose による起動コマンド1つで環境を再現できる |
| NF-31 | ホストに Node.js をインストールせずに動作する |

---

## 6. 外部サービス依存

| サービス | 用途 | 制約 |
|---------|------|------|
| OpenStreetMap | 地図タイル表示 | インターネット接続必須 |
| Nominatim API | 逆ジオコーディング（住所取得） | 1リクエスト/秒の制限、APIキー不要 |

---

## 7. 制約・前提条件

- サーバーは Docker Desktop（WSL2統合有効）が動作する Windows 環境
- データベースは単一 SQLite ファイルで運用（大規模同時接続には不適）
- 認証・認可機能は対象外（社内ネットワーク等での利用を前提）
