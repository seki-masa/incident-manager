## 2. データベース詳細設計

### 2.1 スキーマ（prisma/schema.prisma）

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model Hazard {
  id          String    @id @default(cuid())
  type        String
  title       String
  description String
  lat         Float
  lng         Float
  prefecture  String?
  dangerLevel Int
  status      String    @default("active")
  occurredAt  DateTime
  resolvedAt  DateTime?
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
}
```

- **id:** CUID（衝突確率が極めて低い文字列ID）
- **updatedAt:** Prisma が自動更新。楽観的ロックのキーとして使用
- **type / status:** Prisma/PostgreSQL の enum を使わず String 型で管理（将来の値追加を容易にするため）
- **provider:** SQLite から PostgreSQL に変更（Vercel サーバーレス対応）

### 2.2 インデックス

現時点でカスタムインデックスなし。Prisma が `@id` に主キーインデックスを自動生成。

## 3. バリデーション設計

### 3.1 Zod スキーマ（src/lib/schemas.ts）

```typescript
const HAZARD_TYPES = [
  'earthquake', 'tsunami', 'typhoon', 'flood',
  'fire', 'volcanic', 'military', 'other',
] as const

const HAZARD_STATUSES = ['active', 'monitoring', 'resolved'] as const

// POST /api/hazards 用
export const CreateHazardSchema = z.object({
  type:        z.enum(HAZARD_TYPES),
  title:       z.string().min(1).max(200),
  description: z.string().max(2000).default(''),
  lat:         z.number().min(-90).max(90),
  lng:         z.number().min(-180).max(180),
  prefecture:  z.string().max(100).optional(),
  dangerLevel: z.number().int().min(1).max(5),
  status:      z.enum(HAZARD_STATUSES).default('active'),
  occurredAt:  z.string().datetime(),           // ISO 8601 強制
})

// PUT /api/hazards/:id 用（全フィールド任意）
export const UpdateHazardSchema = z.object({
  type:        z.enum(HAZARD_TYPES).optional(),
  title:       z.string().min(1).max(200).optional(),
  description: z.string().max(2000).optional(),
  prefecture:  z.string().max(100).nullable().optional(),
  dangerLevel: z.number().int().min(1).max(5).optional(),
  status:      z.enum(HAZARD_STATUSES).optional(),
  occurredAt:  z.string().datetime().optional(),
  resolvedAt:  z.string().datetime().nullable().optional(),
})

// GET /api/hazards クエリパラメータ用
export const ListFilterSchema = z.object({
  type:     z.enum(HAZARD_TYPES).optional(),
  status:   z.enum(HAZARD_STATUSES).optional(),
  minLevel: z.coerce.number().int().min(1).max(5).optional(),
  maxLevel: z.coerce.number().int().min(1).max(5).optional(),
})
```

バリデーション失敗時のレスポンス:
```json
{
  "error": "Validation failed",
  "details": {
    "fieldErrors": { "dangerLevel": ["Number must be less than or equal to 5"] },
    "formErrors": []
  }
}
```

## 4. 認証設計

### 4.1 NextAuth 設定（src/lib/auth.ts）

```typescript
export const authOptions: NextAuthOptions = {
  providers: [
    CredentialsProvider({
      credentials: {
        username: { type: 'text' },
        password: { type: 'password' },
      },
      async authorize(credentials) {
        if (
          credentials?.username === process.env.ADMIN_USERNAME &&
          credentials?.password === process.env.ADMIN_PASSWORD
        ) {
          return { id: '1', name: credentials.username }
        }
        return null  // 認証失敗
      },
    }),
  ],
  pages: { signIn: '/login' },
  session: { strategy: 'jwt', maxAge: 24 * 60 * 60 },
  secret: process.env.NEXTAUTH_SECRET,
}
```

### 4.2 ミドルウェア（src/middleware.ts）

すべてのリクエストに対して以下の順で処理する:

```
リクエスト受信
    │
    ├─ 静的アセット (_next/static, _next/image, favicon) → スキップ
    │
    ├─ レート制限チェック（IP ベース、100 req/min）
    │     └─ 超過 → 429 Too Many Requests
    │
    ├─ /api/auth/* または /login → 認証チェックをスキップ
    │
    └─ 認証チェック（JWT トークン検証）
          ├─ 有効 → 処理継続
          ├─ 無効（/api/*） → 401 Unauthorized
          └─ 無効（ページ） → /login へリダイレクト
```

**レート制限実装:**
- 方式: インメモリ スライドウィンドウ（1分間）
- 上限: IP アドレスあたり 100 リクエスト/分
- 注意: サーバーレス環境ではインスタンスをまたいだカウントにはならない（基本的保護として機能）

## 5. API 詳細設計

### 5.1 GET /api/hazards

**ファイル:** `src/app/api/hazards/route.ts`

**クエリパラメータ:** `ListFilterSchema` で検証

| パラメータ | 型 | 説明 |
|-----------|-----|------|
| type | string | 種別でフィルタ（8種別のみ許可） |
| status | string | ステータスでフィルタ（3値のみ許可） |
| minLevel | number | 危険レベル下限 1〜5 |
| maxLevel | number | 危険レベル上限 1〜5 |

**処理フロー:**
1. `ListFilterSchema.safeParse` でクエリパラメータを検証（失敗: 400）
2. Prisma `findMany` で `where` 句を動的構築
3. `occurredAt` 降順でソートして返却（200）
4. DB エラー → 500

---

### 5.2 POST /api/hazards

**ファイル:** `src/app/api/hazards/route.ts`

**処理フロー:**
1. JSON パース（失敗: 400）
2. `CreateHazardSchema.safeParse` で検証（失敗: 400 + エラー詳細）
3. `occurredAt` を `Date` に変換
4. Prisma `create` で登録（成功: 201 / DBエラー: 500）

---

### 5.3 GET /api/hazards/:id

**処理フロー:**
1. `findUnique` で取得
2. 存在しない場合: 404
3. 成功: 200 + Hazard オブジェクト
4. DBエラー: 500

---

### 5.4 PUT /api/hazards/:id

**処理フロー:**

```
JSON パース失敗 → 400
    │
    ├─ updatedAt を本体から分離（楽観的ロック用）
    │
    ├─ UpdateHazardSchema.safeParse → 失敗: 400
    │
    ├─ updatedAt あり？
    │     ├─ findUnique → レコードなし: 404
    │     └─ updatedAt 不一致 → 409 { error: 'conflict', current }
    │
    └─ prisma.hazard.update
          ├─ P2025 (レコード不在) → 404
          ├─ 成功 → 200 + 更新後 Hazard
          └─ その他 DB エラー → 500
```

**特記事項:**
- `updatedAt` は `data` から除外してから `update` を実行（自動更新に任せる）
- `occurredAt` / `resolvedAt` は文字列を `Date` に変換、`resolvedAt: null` も許容

---

### 5.5 DELETE /api/hazards/:id

**処理フロー:**
1. `prisma.hazard.delete` 実行
2. P2025（存在しない）→ 404
3. 成功: 204 No Content
4. その他 DB エラー: 500

### 6.1 page.tsx（HomePage）

**種別:** Client Component  
**役割:** アプリ全体の状態管理・イベントハンドラの定義

#### 状態定義

| 変数名 | 型 | 説明 |
|--------|-----|------|
| hazards | Hazard[] | 全ハザードのリスト |
| selected | Hazard \| null | 詳細パネルで表示中のハザード |
| formCoords | { lat, lng } \| null | 新規登録フォームの表示トリガー兼座標 |
| formPrefecture | string \| null | 逆ジオコーディング結果（null=取得中） |
| notification | string \| null | 通知バナーのメッセージ |
| isEditingRef | RefObject\<boolean\> | 詳細パネルが編集中かのフラグ（再レンダリング不要） |
| selectedRef | RefObject\<Hazard\|null\> | ポーリング時の selected 参照用 |

#### ポーリング処理

```typescript
const POLL_INTERVAL = 30_000  // 30秒

useEffect(() => {
  const timer = setInterval(async () => {
    const fresh = await fetch('/api/hazards').then(r => r.json())
    setHazards(fresh)
    // 編集中でなければ selected を最新版に同期
    if (!isEditingRef.current && selectedRef.current) {
      const updated = fresh.find(h => h.id === selectedRef.current.id)
      setSelected(updated ?? null)  // 削除されていれば null（パネルを閉じる）
    }
  }, POLL_INTERVAL)
  return () => clearInterval(timer)
}, [fetchHazards])
```

#### 逆ジオコーディング処理

```typescript
useEffect(() => {
  if (!formCoords) { setFormPrefecture(null); return }
  const controller = new AbortController()

  fetch(`https://nominatim.openstreetmap.org/reverse?format=json&lat=${formCoords.lat}&lon=${formCoords.lng}&accept-language=ja`,
    { signal: controller.signal }
  )
  .then(r => r.json())
  .then(data => {
    const addr = data.address ?? {}
    const state = addr.state
      || data.display_name?.split(',').map(s => s.trim()).find(p => /[都道府県]$/.test(p))
      || ''
    const city  = addr.city ?? addr.municipality ?? addr.town ?? addr.village ?? addr.county ?? ''
    const ward  = addr.city_district ?? addr.suburb ?? ''

    // Set で重複排除してから結合
    const seen = new Set<string>()
    const parts: string[] = []
    for (const part of [state, city, ward]) {
      if (part && !seen.has(part)) { seen.add(part); parts.push(part) }
    }
    setFormPrefecture(parts.join(''))
  })
  .catch(() => setFormPrefecture(''))

  return () => controller.abort()
}, [formCoords])
```

**住所フォールバック戦略:**

| 住所レベル | 第1候補 | 第2候補以降 | 備考 |
|-----------|---------|------------|------|
| 都道府県 | `addr.state` | `display_name` から `/[都道府県]$/` で抽出 | 東京23区で state が空のケース対応 |
| 市・町・村 | `addr.city` | `municipality` → `town` → `village` → `county` | - |
| 区 | `addr.city_district` | `addr.suburb` | 政令指定都市・東京23区で異なる |

#### handleUpdate（楽観的ロック）

```typescript
const handleUpdate = async (id: string, data: object, updatedAt?: string) => {
  const res = await fetch(`/api/hazards/${id}`, {
    method: 'PUT',
    body: JSON.stringify({ ...data, ...(updatedAt && { updatedAt }) })
  })

  if (res.status === 409) {
    const { current } = await res.json()
    throw Object.assign(new Error('conflict'), { current })
  }
  if (res.status === 404) {
    setHazards(prev => prev.filter(h => h.id !== id))
    setSelected(null)
    setNotification('このハザードは別のユーザーによって削除されました')
    throw new Error('deleted')
  }
  if (!res.ok) throw new Error('update_failed')

  const updated = await res.json()
  setHazards(prev => prev.map(h => h.id === id ? updated : h))
  setSelected(updated)
}
```

### 6.2 HazardDetailPanel.tsx

**種別:** Client Component  
**役割:** ハザード詳細表示・インライン編集・競合解消

#### 状態定義

| 変数名 | 型 | 説明 |
|--------|-----|------|
| editing | boolean | 編集モードのトグル |
| form | FormState | 編集フォームの値（type, title, description, prefecture, dangerLevel, status, occurredAt） |
| saving | boolean | 保存中フラグ（ボタン無効化に使用） |
| conflictHazard | Hazard \| null | 409 受信時のサーバー最新データ |

#### 競合解消フロー

```
handleSave()
 └─ onUpdate(id, payload, hazard.updatedAt)  ← updatedAt 付きで送信
      ├─ 成功 → setEditing(false)
      ├─ 409 Conflict → setConflictHazard(current)  ← 競合UIを表示
      ├─ 404 Not Found → page.tsx が通知・パネルを閉じる
      └─ その他エラー → 何もしない（エラーは page.tsx で通知済み）

競合UI表示後のユーザー選択:
 ├─ "最新版を確認"   → onConflict(conflictHazard), setEditing(false)
 └─ "強制上書き保存" → onUpdate(id, payload)  ← updatedAt なし（競合チェックなし）
```

#### Props インターフェース

```typescript
interface Props {
  hazard: Hazard | null
  onClose: () => void
  onUpdate: (id: string, data: object, updatedAt?: string) => Promise<void>
  onStatusChange: (id: string, status: HazardStatus) => void
  onEditingChange?: (editing: boolean) => void  // isEditingRef 更新用
  onConflict: (current: Hazard) => void
}
```

### 8.1 APIエラー対応表

| HTTPステータス | 発生場面 | クライアント側処理 |
|--------------|---------|-------------------|
| 400 | バリデーション失敗 | エラー詳細をログ出力（UI未表示） |

| 500 | 予期しないサーバーエラー | 汎用エラーメッセージのみ返却（内部情報は非公開） |

### 9.2 セキュリティヘッダー一覧

| ヘッダー | 値 | 目的 |
|---------|-----|------|
| X-Content-Type-Options | nosniff | MIMEスニッフィング防止 |
| X-Frame-Options | DENY | クリックジャッキング防止 |
| X-XSS-Protection | 1; mode=block | レガシーブラウザ向け XSS 防止 |
| Referrer-Policy | strict-origin-when-cross-origin | リファラ情報の制限 |
| Permissions-Policy | camera=(), microphone=(), geolocation=() | 不要なブラウザ機能を無効化 |
| Content-Security-Policy | （下記参照） | リソース読み込み元を制限 |

**CSP 設定:**

| ディレクティブ | 許可元 | 理由 |
|-------------|-------|------|
| default-src | 'self' | デフォルトは同一オリジンのみ |
| script-src | 'self' 'unsafe-eval' 'unsafe-inline' | Next.js HMR・React DevTools に必要 |
| style-src | 'self' 'unsafe-inline' | Leaflet のインラインスタイルに必要 |
| img-src | 'self' data: blob: https://*.tile.openstreetmap.org | 地図タイル画像 |
| connect-src | 'self' https://nominatim.openstreetmap.org | 逆ジオコーディング API |
| frame-src | 'none' | iframe 埋め込みを完全禁止 |
| object-src | 'none' | Flash 等のオブジェクト埋め込みを禁止 |
